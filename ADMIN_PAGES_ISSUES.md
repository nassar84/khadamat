# إصلاح صفحات التحكم في المجموعات والخدمات

## المشاكل المحتملة

### 1. صفحة إدارة التصنيفات (Categories Management)

**المشكلة**: استخدام `prompt()` و `confirm()` البسيطة بدلاً من نماذج منبثقة احترافية

**الحل**: إنشاء نماذج منبثقة (Modals) احترافية مع حقول إدخال كاملة

#### الوظائف الحالية:
- ✅ عرض التصنيفات الرئيسية
- ⚠️ إضافة تصنيف رئيسي (يستخدم prompt بسيط)
- ⚠️ تعديل تصنيف رئيسي (يستخدم prompt بسيط)
- ✅ حذف تصنيف رئيسي
- ⚠️ إضافة تصنيف فرعي (يستخدم prompt بسيط)
- ✅ حذف تصنيف فرعي

#### ما يحتاج إلى تحسين:
1. استبدال `prompt()` بنماذج منبثقة احترافية
2. إضافة حقول لتحديد الأيقونة واللون
3. إضافة حقل الترتيب (Order)
4. معالجة الأخطاء بشكل أفضل

### 2. صفحة إدارة الخدمات (Services Management)

**المشكلة**: الصفحة للعرض فقط، لا توجد وظائف للتعديل أو الحذف

#### الوظائف الحالية:
- ✅ عرض قائمة الخدمات
- ✅ الترقيم (Pagination)
- ✅ عرض حالة الموافقة
- ❌ زر "إضافة خدمة" غير متصل بأي وظيفة
- ❌ قائمة الإجراءات (Dropdown) غير متصلة بأي وظائف

#### ما يحتاج إلى تنفيذ:
1. ربط زر "إضافة خدمة" بصفحة إنشاء خدمة
2. تفعيل خيارات القائمة المنسدلة:
   - عرض التفاصيل
   - تعديل
   - حذف
   - تغيير حالة الموافقة

## الحلول المقترحة

### الحل 1: تحسين صفحة التصنيفات

سأقوم بإنشاء نماذج منبثقة احترافية لـ:
1. إضافة/تعديل تصنيف رئيسي (مع حقول: الاسم، الأيقونة، اللون، الترتيب)
2. إضافة/تعديل تصنيف فرعي (مع حقول: الاسم، التصنيف الرئيسي)
3. إضافة/تعديل تخصص (SubCategory)

### الحل 2: تحسين صفحة الخدمات

سأقوم بـ:
1. ربط زر "إضافة خدمة" بصفحة `/provider/create-service`
2. تفعيل وظائف القائمة المنسدلة:
   - عرض: الانتقال إلى `/service/{id}`
   - تعديل: الانتقال إلى `/provider/edit-service/{id}`
   - حذف: حذف الخدمة مع تأكيد
   - الموافقة/الرفض: استخدام AdminService

## الخطوات التالية

1. ✅ تحديد المشاكل
2. ⏳ إنشاء نماذج منبثقة للتصنيفات
3. ⏳ تفعيل وظائف صفحة الخدمات
4. ⏳ اختبار جميع الوظائف

## ملاحظات مهمة

- جميع الـ API Endpoints موجودة وتعمل بشكل صحيح
- المشكلة في الواجهة الأمامية (Frontend) فقط
- يتطلب التحديث إضافة نماذج منبثقة وربط الأزرار بالوظائف المناسبة
